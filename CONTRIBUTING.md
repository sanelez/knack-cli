# Contributing

Thanks for the interest. Knack is maintained by a small team and we want this
repo to stay easy to ship from.

## What we accept

- **Bug fixes**: gladly, especially platform-specific quirks (Windows path
  handling, `gh` version differences, etc).
- **Documentation fixes**: typos, clarifications, broken links. Send a PR.
- **Security reports**: email security@getknack.ai. Please do not open public
  issues for vulnerabilities.

## What we ask you to discuss first

- New commands or subcommands
- New flags on existing commands
- New backends or auth drivers
- Refactors of more than one file

Open a Discussion or an Issue describing what you want to change and why.
We will respond. Once we agree on the shape, a PR is welcome.

## Why this norm

Knack is a small, focused tool. We want to keep the surface area tight so it
stays predictable for users and reviewable for maintainers. The "talk first,
code later" rule is the cheapest way to avoid wasted work on both sides.
